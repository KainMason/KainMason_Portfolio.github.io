CS-499 Progress (Milestone Two)

For Milestone Two, I focused on enhancing the Travlr Getaways project as my Software Design & Engineering artifact. The goal was not just to have working code, but to improve the structure, maintainability, and clarity of the application so it looks and feels more like something used in a real development environment.

Key Enhancements Completed

  • Refactored backend structure: Separated routing, controllers, and configuration into clearer folders and removed duplicated logic to make the API easier to follow and maintain.
  • Improved error handling: Added more consistent API responses and status codes so failures are easier to debug and less confusing to clients.
  • Cleaned up MongoDB queries: Standardized query patterns and removed unused or redundant queries to keep the data access layer predictable.
  • Enhanced documentation: Expanded the README with setup steps, technology stack details, and a short architecture overview so a new developer or hiring manager can quickly understand how the system fits together.
  • Front-end layout tweaks: Made small consistency improvements to the customer-facing pages to keep the UI more readable and aligned across views.

These changes support the CS-499 outcomes around software engineering/design, communication, and developing a security-aware mindset. I am still iterating, but this represents my current level of progress in the course.

Across all projects, I focus on writing clear, maintainable code, documenting my decisions, and building solutions that would actually be usable in a real environment, not just as class assignments.
